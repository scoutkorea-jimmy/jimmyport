# Jimmy Park Personal Website Guide

> Version: 1.0  
> Purpose: Personal profile website / collaboration introduction / official search profile  
> Primary User: Jimmy Park / 박지민  
> Design Mood: Clean · Modern · Lightweight · Professional  

---

## 1. Project Overview

This website is not an art portfolio or a heavy corporate website.  
It is a personal profile and work-introduction website for Jimmy Park / 박지민.

The primary goal is to answer the following questions quickly:

- Who is Jimmy Park?
- What kind of work does he do?
- What kind of collaboration can be proposed to him?
- Where can people find an official profile or contact point for him?

The site should work as:

1. A personal profile site.
2. A collaboration inquiry landing page.
3. A search-friendly official introduction page for Naver, Google, and social sharing.
4. A lightweight international-facing profile that can be shared with domestic and overseas contacts.

## 6. Design Direction

### 6.1 Overall Mood

| Keyword | Direction |
|---|---|
| Clean | Remove unnecessary decoration. |
| Modern | Use refined spacing, typography, and card systems. |
| Lightweight | Avoid heavy media, unnecessary animation, and video backgrounds. |
| Professional | Make the site suitable for collaboration inquiries. |
| Field-based | Use real field images rather than staged studio imagery. |
| Global-friendly | Make it easy for both Korean and international visitors to understand. |
| Direct | Keep navigation and content structure intuitive. |

---

### 6.2 What to Avoid

Avoid the following:

- Art portfolio mood.
- Overly emotional self-branding.
- Heavy corporate website structure.
- Complex developer portfolio style.
- Excessive animation.
- Fullscreen video backgrounds.
- Too many navigation items.
- Overly Scout-like institutional design.
- Too much philosophical explanation on the first screen.

---

## 7. Visual Style Guide

### 7.1 Color Palette

Recommended palette:

| Use | Color | Hex |
|---|---|---|
| Main Background | White | `#FFFFFF` |
| Soft Background | Off-white | `#F7F7F5` |
| Main Text | Charcoal | `#111111` |
| Secondary Text | Gray | `#666666` |
| Border | Light Gray | `#E5E5E5` |
| Primary Accent | Burgundy | `#7A1E2C` |
| Optional Sub-accent | Forest Green | `#1F4D3A` |

Usage principles:

- Use white and off-white as the base.
- Use charcoal for primary text.
- Use burgundy as the main accent.
- Use forest green only as a subtle Scouting-related sub-accent.
- Do not make the whole site look like an official Scout organization website.

---

### 7.2 Typography

Recommended style:

| Element | Direction |
|---|---|
| Korean font | Pretendard, Noto Sans KR, or system sans-serif |
| English font | Inter, Helvetica, or system sans-serif |
| Headings | Bold, clear, restrained |
| Body | Highly readable, moderate line height |
| Buttons | Simple, medium weight |
| Decorative fonts | Avoid |

Suggested hierarchy:

| Type | Size Direction |
|---|---|
| H1 | 48–64px desktop / 34–42px mobile |
| H2 | 28–36px desktop / 24–30px mobile |
| Body | 15–17px |
| Caption | 12–14px |
| Button | 14–16px |

---

### 7.3 Layout

| Element | Direction |
|---|---|
| Max width | 1100–1280px |
| Grid | Simple 12-column or flexible card grid |
| Spacing | Generous whitespace |
| Sections | Clear hierarchy with subtle dividers |
| Cards | Clean white cards with light border |
| Corners | Slightly rounded, not overly playful |
| Shadows | Minimal or none |
| Mobile | Mobile-first responsive layout |
| Animation | Minimal fade or slide only if necessary |

## 9. Work Page Content Direction

### 9.1 Photography

This section should not look like an art photography gallery.

Core message:

> Jimmy Park mainly works with event sketch, field documentation, and purpose-based photography. He can identify necessary scenes on site and provide press-ready photos quickly, including same-day delivery when required.

Key focus:

| Focus | Description |
|---|---|
| Event Sketch | Capture overall atmosphere and key moments of events |
| Field Documentation | Record people, context, flow, and details |
| Press Photo | Select and deliver usable photos for press or SNS |
| Same-day Delivery | Provide selected and edited photos quickly on site |
| Purpose-based Images | Prepare different image types depending on use |

Use cases:

- Press releases.
- SNS posts.
- Homepage updates.
- Card news.
- Reports.
- Archives.
- Internal documentation.

Visual direction:

- Event venue.
- Speakers.
- Participants.
- Networking.
- Group photos.
- Detail shots.
- Backstage moments.

Avoid:

- Fine-art gallery tone.
- Overly dramatic photo captions.
- Unnecessary long explanations.

---

### 9.2 Video

Core message:

> Jimmy Park designs and produces video formats according to the purpose of the project.

Video categories:

| Category | Description |
|---|---|
| Promotion | Brand, institution, campaign promotion |

| Interview | Interview-based storytelling |
| IR / PR | Institutional or startup communication |
| Campaign | Purpose-driven message campaign |

Visual direction:

- Video thumbnails.
- Camera and production setup.

Avoid:

## 17. Claude Design Prompt

Use the following prompt to generate the design:

```text
Design a clean, modern, lightweight personal profile website for Jimmy Park / 박지민.

This website is not an art portfolio and not a corporate company website. It is a personal profile and collaboration-introduction site that explains who Jimmy Park is, what he does, and how people can contact him for collaboration. It should also work as an official search-friendly profile page for Naver, Google, and international visitors.

The core activity areas are Photography, Video, Scouting, and Vibe Coding. Keep the navigation simple: Home, Work, Scouting, Contact. The Admin page should exist at /admin but must be hidden from public navigation.

The design mood should be clean, modern, lightweight, professional, field-based, global-friendly, and direct. Use generous whitespace, strong but restrained typography, simple cards, clear hierarchy, and field-oriented imagery. Avoid heavy animation, fullscreen video backgrounds, overly artistic gallery moods, complex developer portfolio styling, or corporate presentation-like layouts.

Use a color palette based on white, off-white, charcoal, gray, light gray, and burgundy as the main accent. Forest green may be used only as a subtle Scouting-related sub-accent. Use modern sans-serif typography such as Pretendard, Inter, Noto Sans KR, or system fonts.

Home should begin with a simple hero section: Jimmy Park / 박지민, a short headline, a concise intro, View Work and Contact buttons, and one strong field-based hero image. The homepage should include four cards for Photography, Video, Scouting, and Vibe Coding; a Featured Work section; a Now / Current Roles section; and a final contact CTA.

Work should include Photography, Video, and Vibe Coding in one page. Photography should focus on event sketch, field documentation, press-ready photos, same-day delivery, and purpose-based image selection. Video should focus on purpose-based video categories such as promotion, event film, interview, IR/PR, campaign, and short-form. Vibe Coding should focus on AI-assisted prototypes, web tools, campaign pages, and content operation systems. Do not make Vibe Coding look like a heavy developer portfolio.

Scouting should have its own page because it is a core identity and trust asset. It should present international Scouting experience, youth movement communication, Jamboree media work, BP Media, Scout Tour Assistant, Jamboree D-count, key roles, timeline, projects, and selected field images. Keep it consistent with the personal site design and avoid making it look like an official Scout organization website.

Contact should be very simple: collaboration fields, email, SNS links, Korean/English availability, and a concise inquiry message.

Also design a hidden admin page at /admin. The admin should include login-only access and allow Jimmy Park to manage Home, Work, Scouting, Projects, Media Library, Contact, and SEO. Admin functions should include upload, edit, delete, hide/show, sort order, image management, video link management, project card management, and SEO field editing.

Prioritize fast loading, mobile-first responsive design, global access, clear search visibility, and easy content management.
```
