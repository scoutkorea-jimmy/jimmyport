# Jimmy Park Website — Final Revision Guide

> Version: 2.0  
> Purpose: English-first personal profile website revision guide  
> Target: Claude Design  
> Site Owner: Jimmy Park / 박지민  
> Core Direction: Purpose-based profile site, not a flat portfolio  

---

## 1. Revision Objective

The current design direction is acceptable.

Keep the following mood:

- Clean
- Modern
- Lightweight
- Warm-minimal
- Professional
- Soft rounded corners
- Off-white background
- Burgundy accent
- Subtle forest green only for Scouting

However, the current site still feels too flat.

Jimmy Park is being presented as four separate categories:

```text
Photography
Video
Scouting
Vibe Coding
```

This makes him feel like a list of skills rather than a layered person with a clear working method.

The revised website should present Jimmy Park as someone who:

1. Understands the purpose of a project.
2. Suggests the most suitable direction.
3. Documents, produces, connects, or builds what is needed.
4. Delivers practical outputs that can actually be used.

The site must shift from:

```text
“I do four different things.”
```

to:

```text
“Tell me the purpose. I will suggest the right direction and execute what needs to be made.”
```

---

## 2. Language Direction

The website should be English-first.

Korean should be used only as:

- short secondary support text,
- small labels,
- captions,
- Korean nuance below English headings.

Do not make Korean equal in hierarchy to English.

### Correct Direction

```text
Field Documentation
현장 기록

Purpose-based Production
목적 기반 제작

Scouting Communication
스카우팅 커뮤니케이션

AI-assisted Prototyping
AI 기반 프로토타이핑
```

### Avoid

- Long Korean paragraphs on the main pages.
- Korean and English competing at the same visual level.
- Full bilingual duplication of every section.
- Korean-first structure with English added later.

English should lead the structure.  
Korean should quietly support the meaning.

## 12. Recommended Interactions

The site should have subtle, purposeful interactions.  
Do not use flashy portfolio animations.

### 12.1 Header Interaction

- Header remains minimal.
- On scroll, header becomes slightly more compact.
- Active page indicator uses burgundy underline or soft background.
- Contact button remains visible.

---

### 12.2 Hero Image Interaction

- Hero image may have subtle parallax or slow reveal.
- Image caption appears softly on load.
- No aggressive motion.

---

### 12.3 Activity Area Interaction

For the four activity areas:

- On hover, reveal a second line showing practical outputs.

Examples:

```text
Photography → Press · SNS · Report · Archive
Video → Promotion · Interview · Event Film
Scouting → Youth · International · Media
Vibe Coding → Campaign · Map · Content Tool
```

---

### 12.4 Project Card Interaction

Cards should slightly lift or reveal details on hover.

```text
Category

Status
```

Do not use heavy scaling.

### 12.5 Approach Section Interaction

- Use horizontal step flow on desktop.

- On mobile, use vertical timeline.
- Keep animation subtle.
